# Бриф проекта: предиктивная аналитика состояния крутильного вала (МоДеРо)

> Краткий единый источник правды по проекту. Используется как контекст для исследовательского чата.
> Статус: подход и пайплайн утверждены; открыты 3 вопроса (см. раздел «Открытые вопросы»).

## 1. Контекст и цель
- **Предметная область:** турбоагрегат на предприятии; диагностируем состояние **крутильного вала** в переходных режимах по работе нагрузки.
- **Тип работы:** учебно-/научно-исследовательская (УИР/НИР), с пояснительной запиской (ПЗ) по требованиям кафедры.
- **Целевая переменная:** `F1` — 1-я собственная частота крутильных колебаний вала (1-я СЧ ККВ), Гц. Главный диагностический показатель.
- **Цель:** модель предиктивной аналитики, которая (1) строит **собственную** режимную модель оценки F1 и фиксирует её точность (Prism/ДП1 — справочный контекст, не цель обогнать), (2) строит устойчивый индикатор «здоровья» вала по остаткам с порогами, (3) прогнозирует индикатор и F1 на горизонт (упреждение). Отдельно — поведение на переходных режимах.

## 2. Данные
**Файл A (сырой, основной для обучения):** `Данные_ПТК_МоДеРо_за_2015-2016гг.xlsx`
- ~12 470 почасовых строк, 04.03.2015 → 18.01.2017 (≈685 дней), пропусков в F1/признаках нет.
- 34 разрыва >2 ч (резать на непрерывные сегменты, не интерполировать через дыры).
- Агрегат работает непрерывно: Pa никогда < ~36 МВт → **полных пусков/остановов нет**.

**Файл B (обработанный, для бенчмарка):** `Prism_МоДеРо.xlsx`
- 6 679 строк, 2016–2017 (подпериод файла A; F1 совпадает с A точно) + остатки эталонных моделей **Prism** (Klinkman) и **ДП1** (предыдущий диплом).
- Остаток = F1 − прогноз модели. Остатки Prism/ДП1 — **справочный контекст для сравнения**, не целевая планка (для ориентира: RMSE остатков ДП1 ≈ 0.0021 Гц).

**Ключевые свойства F1 и режима:**
- F1 почти постоянна: среднее ≈ 19.31 Гц, σ ≈ 0.016, размах 19.26–19.38.
- F1 коррелирует с режимом: Pr ≈ 0.49, Tst ≈ 0.43, Pa ≈ 0.41; с Fr (частота вращения, ~50 Гц) — нет.
- Автокорреляция F1: 0.87 (1 ч), 0.37 (6 ч), **0.49 (24 ч)** — есть суточный отклик (нагрузка следует суточному графику); недельная (168 ч) слабая (0.11).
- Naive-persistence (F1(t+h)=F1(t)): RMSE ≈ 0.006 Гц (1 ч), ≈ 0.015 Гц (6 ч) — сильный baseline.
- **Переходные режимы:** |ΔPa|>20 МВт ≈ 18% часов; в них F1 «гуляет» в ~2.5 раза сильнее (медиана |ΔF1| ≈ 0.0075 против 0.0030 Гц в установившемся). Самое трудное и информативное — переходы.

**Признаки:** `Pa` (акт. мощность, МВт), `Pr` (реакт. мощность, МВАр), `Td`/`Tg`/`Tst` (температуры дистиллята/пара/статора, °C), `Тп.ЦНД-1/2`, `Т ГПЗ н.А–Г`, `Т турб. н.А–Г`, `Fr` (частота вращения, Гц). Полные определения — в отдельной таблице признаков.

## 3. Утверждённый подход — Вариант C (гибрид)
Идея: не прогнозировать F1 «в лоб» (плохо поставлено — F1 диктуется будущим режимом), а **разложить**:

`F1(t) = f_reg(x(t)) + r(t)`
- `f_reg(x(t))` — режимная (объяснимая) часть: F1 как функция нагрузки/температур.
- `r(t) = F1 − f_reg` — остаток-индикатор «здоровья»: устойчивый дрейф = потеря жёсткости = ранний признак дефекта.

**Три слоя:**
1. **Слой 1 — модель режима** `f_reg`: регрессия F1 по режиму. Цель — **своя зафиксированная модель**; persistence/Prism/ДП1 — для сравнения и контекста, не мишень.
2. **Слой 2 — индикатор здоровья** `H(t)`: сглаживание остатка (EWMA) + CUSUM + контрольные пороги.
3. **Слой 3 — прогноз на горизонт:** (a) прогноз `H(t+h)` → запас до порога/упреждение; (b) `F1̂(t+h)=f_reg(плановый режим)+Ĥ(t+h)`.

## 4. Технический пайплайн (этапы)
1. **ETL** — сборка единой почасовой таблицы из A+B, согласование меток, перевод запятых-десятичных, сегментация по разрывам, флаг переходного часа.
2. **EDA** — распределения, связи F1↔режим, тренд/автокорреляция, «установившийся vs переходный».
3. **Признаки** — режимные + лаги (внутри сегментов) + скорость рампы |ΔPa| + скользящие.
4. **Слой 1** — линейная (baseline) + градиентный бустинг (CatBoost/LightGBM); сравнение с persistence/Prism/ДП1.
5. **Слой 2** — индикатор `H(t)`, пороги, логика тревоги.
6. **Слой 3** — прогноз `H(t+h)` и `F1̂(t+h)`; rolling-origin бэктест.
7. **Валидация детектора** — впрыск синтетического дрейфа, проверка обнаружения CUSUM (задержка vs скорость дрейфа).
8. **Под-исследование переходов** — метрики и поведение остатка steady vs transient.
9. **Артефакты** — рисунки/таблицы/схема + модульное оформление кода.

## 5. Разбиение по времени (без перемешивания — иначе утечка)
- Обучение 03.2015–06.2016 · Валидация 07–10.2016 · Тест 11.2016–01.2017.
- Прогноз — rolling-origin бэктест. Опорное «здоровое» окно для порогов CUSUM — стабильный ранний участок.
- Перекрытие с Prism/ДП1 (2016–2017) — только для сравнения, не для обучения порогов.

## 6. Инструменты и метрики
- **Стек:** Python — pandas/numpy, scikit-learn, CatBoost/LightGBM, statsmodels, matplotlib.
- **Метрики:** слой 1 — RMSE, MAE, R² (vs persistence/Prism/ДП1); слой 3 — RMSE по горизонтам, упреждение, частота ложных тревог на здоровых данных, задержка обнаружения на синтетическом дрейфе.

## 7. Оговорки и риски (заложены сразу)
- **Нет размеченных отказов** → детектор валидируем синтетическим дрейфом, не реальными авариями (подаём как методологию).
- F1 почти постоянна → persistence силён; акцент на режимной модели и индикаторе, а не на «голом» прогнозе.
- Прогноз F1 по варианту (b) опирается на наличие плана нагрузки.
