### Data hygiene — две найденные дефекта Prism-файла (материал для практ. раздела ПЗ)

1. **Псевдо-нули Prism = «не посчитано».** `resid_prism` валиден лишь в **1219** из 6 679 часов перекрытия; остальные 5 460 — ровно `0` (разбросаны). Маппинг `0 → NaN`. Иначе RMSE Prism ложно занижается до 0.0021 Гц вместо честных **0.0050 Гц** на N=1219.
2. **«ДП1» = Prism × 517.7169** (константа, std=0). Не независимая модель, а масштабированная копия. ДП1 как отдельный baseline **исключён**. В сравнении остаются: **persistence** (везде) и **Prism** (на N=1219 ч).

## 3. Этап 3 — признаки  ✅

**Артефакт:** `outputs/modero_features.parquet` (12 470 × 74, 68 признаков).

| группа | состав | n |
|---|---|---|
| режимные (passthrough) | 16 (Pa, Pr, T·, Fr) | 16 |
| рампы | `d{Pa,Pr}_{1,3,6}h` + `abs_…` | 12 |
| лаги | F1 ×[1,6,24]; драйверы ×[1,24] | 15 |
| скользящие | {Pa,Pr,Tst}×{3,6,24} mean+std; F1×{6,24} mean+std (сдвиг 1) | 22 |
| календарные | hour_sin/cos, dow | 3 |

**Правила без утечки** (все проверены):
- Лаги/рампы/скользящие — **только внутри сегментов** (`groupby('seg')`). На стартах сегментов → NaN, прошлый seg не подтягивается.
- Скользящие по F1 — **со сдвигом 1** (только прошлое, цель не утекает).
- Скользящие по драйверам — включают текущий час (режим в `t` известен).

Доля NaN в признаках — 1.44% средняя, 95.3% строк полностью валидны.

### Абляция: тепловая динамика  (документированный отрицательный результат)

`features.py --thermal` → `outputs/modero_features_thermal.parquet` (107 признаков). Добавляет лаги/скользящие/EWMA(halflife=24ч)/накопление(72ч) для температурных драйверов + агрегаты Tgpz_mean / Tturb_mean.

**Результат:** test R² CatBoost 0.042 → 0.036 (не улучшил), LinReg val R² −0.42 → −0.69 (переобучение). → дрейф остатка **НЕ сводится** к несмоделированной тепловой инерции. В рабочую f_reg тепло не входит. Метрики сохранены в `outputs/ablation_thermal_layer1_metrics.csv`.

## 4. Этап 4 — слой 1: режимная модель f_reg  ✅

**Модули:** `src/eval.py`, `src/models/baseline.py`, `src/models/gbm.py` (основная).
**Артефакты:** `outputs/layer1_metrics.csv`, `outputs/modero_residuals.parquet`, `outputs/layer1_pred_vs_actual.png`, `outputs/layer1_residual_timeseries.png`.

**Ключевое методологическое решение:** f_reg использует **только режимные признаки** (`regime_feature_columns` исключает все `F1_*`). Иначе медленный дрейф F1 = диагностический сигнал — впитывается в f_reg через лаги F1 и исчезает из остатка → слой 2 ослеп бы. F1-лаги/окна оставлены для persistence-эталона и слоя 3.

**Разбиение по времени (раздел 5 брифа, без shuffle):**
- train: 03.2015 – 06.2016  (n = 8 920)
- val:   07.2016 – 10.2016  (n = 2 718)
- test:  11.2016 – 01.2017  (n =   832)

### Метрики, subset = all

| модель | split | RMSE, Гц | R² |
|---|---|---|---|
| **CatBoost** f_reg | train | 0.01218 | **+0.402** |
| **CatBoost** f_reg | val | 0.01512 | −0.160 |
| **CatBoost** f_reg | test | 0.01434 | +0.042 |
| LinReg baseline | train | 0.01058 | +0.549 |
| LinReg baseline | val | 0.01675 | −0.424 |
| LinReg baseline | test | 0.01567 | −0.248 |
| *persistence (контекст)* | test | 0.00601 | +0.832 |
| *Prism (контекст, N=136)* | test | 0.00469 | — |

### Интерпретация (важная)
- Train R² 0.40 → f_reg **не константа**, ловит реальную структуру режим→F1.
- Test R² ≈ 0 → **нестационарность связи режим→F1** (концепт-дрейф) — это и есть предмет исследования.
- Persistence (R²=0.83) и Prism (RMSE 0.0047) **выше нашей f_reg** — по брифу они **контекст, не цель** (persistence эксплуатирует автокорреляцию 0.87, Prism — откалиброванная промышленная модель).
- Декомпозиция Варианта C осмысленна: остаток несёт несведённую к режиму вариацию.

## 5. Этап 5 — слой 2: индикатор здоровья H(t)  ✅

**Модуль:** `src/health.py`.
**Артефакты:** `outputs/health_series.parquet`, `outputs/health_indicator.png` (EWMA с пределами + CUSUM± с порогом + разметка in/out-envelope/тревог).

### Три методологические корректировки (все согласованы)

**(а) Флаг `in_envelope` — решающий тест природы дрейфа.**
Ключевые драйверы Tcnd1/2, Pa в пределах [min,max] train-огибающей.
- val/test ~100% in-envelope.
- Сдвиг авг–сен 2016: **1257/1258 ч внутри огибающей**, лишь 1 час снаружи.
- → дрейф **легитимен**, это смена связи режим→F1, а не зажим деревьев на экстраполяции. Тревоги засчитываются **только in-envelope**.

**(б) Пороги — по OUT-OF-SAMPLE здоровому окну.**
Опора = `resid_oos_train` (блочная 5-fold CV на train, in-envelope), μ₀=−0.00098, σ=0.01178.
Проверка занижения: OOS-σ (0.0118) ≈ in-sample σ (0.0122) — занижения нет (CatBoost рано остановлен на 26 итерациях).

**(в) Эмпирическая калибровка под FA=1%.**
**⚠️ Поймана ловушка:** классические `L·σ` пределы EWMA/CUSUM дали 77% ложных тревог — остаток сильно автокоррелирован (F1 autocorr 0.87@1ч), формула предполагает iid. Заменил на квантили распределения индикатора на здоровом ряду.

### Параметры

| параметр | значение |
|---|---|
| EWMA α | 0.03 (halflife ≈ 23ч) |
| CUSUM slack k | 0.5·σ = 0.0059 Гц |
| EWMA-предел (эмпир., FA=1%) | ±0.0192 Гц |
| CUSUM-порог h (эмпир., FA=1%) | 0.0589 Гц |
| FA-факт на опоре | **1.1%** ✓ |

### Срабатывания

| период | первая тревога | характер |
|---|---|---|
| train (опора) | — | CUSUM 0%, EWMA 1.1% (откалиброванный FA) |
| val (07–10.2016) | **2016-08-31 15:00** | начало эпизода авг–сен 2016 |
| test (11.2016–01.2017) | непрерывно с 11.01 | CUSUM защёлкнут на устойчивом дрейфе |

**CUSUM** — надёжное правило (0% FA на train, ловит малый устойчивый сдвиг накоплением).
**EWMA** — вспомогательное (точечно шумит из-за автокорреляции, здоровые всплески до +0.053).

### Зафиксированное ограничение (в отчёт)
Начало эпизода **совпадает с летом** (горячая охлаждающая вода, Tcnd1 mean 29.7 vs train 18.3). Корреляция остатка с Tcnd слабая (~0.18 помесячно). На наблюдательных данных **строго отделить «здоровье» от сезона** в этом единственном эпизоде нельзя. Поэтому валидация детектора **синтетическим дрейфом** (этап 7) обязательна.
